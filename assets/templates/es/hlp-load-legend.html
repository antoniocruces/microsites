Estructura

Microsites utiliza un archivo remoto en formato SQLite v3, generado por ExpoFinder y guardado en el servidor de aplicación, HD+. Las tablas que lo componen sirven a diferentes propósitos. Se muestra su estructura para permitir operar con ellas desde la opción SQL o utilizando SQLab. Se muestra el nombre de cada tabla y su descripción, así como la lista de campos que la componen y su tipo.

Leyenda

  • Tipos de campo: INTEGER = número entero; VARCHAR = texto de longitud variable; DATE = fecha; NOT NULL = no acepta valores nulos.
  • Prefijos: start = inicial; end = final; age = edad.
  • Sufijos: y = año; m = mes; d = día; w = semana.
  • Campos utilitarios: ID = número ID; RID = número ID relacionado; rtype/idtype = tipo de registro; ridtype = tipo de registro relacionado; rkey = clave de registro; value = valor de registro.

Tablas especiales

  • FLT. Filtro. Cada colección dispone de uno independiente. Todos los registros de la misma deben incluir en ella su ID.
  • REL. Relaciones de primer grado. Del tipo ID reg. A → tipo rel. A-B → ID reg. B
  • LNK. Relaciones de segundo grado. Del tipo ID reg. A → tipo rel. A-B → ID reg. B → tipo rel. B-C → ID reg. C
    NOTA: las relaciones A-B y B-C, que aparecen en color gris, no se muestran.
